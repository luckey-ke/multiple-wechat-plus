# 微信多开 Plus — v2.0 重构方案

## 一、背景

当前版本存在以下问题：
- **5 个分散入口**：配置、快速启动、多开列表、确认登录、删除列表，需要记不同关键词
- **操作割裂**：每次使用需要在多个界面间来回切换
- **代码维护困难**：6 个 feature 各自独立，逻辑重复
- **存在多个 bug**：路径硬编码、回调错误、搜索失效等

## 二、重构目标

1. **统一界面**：一个入口，一个界面，完成所有操作
2. **简化交互**：点击卡片即可切换账号，无需记忆关键词
3. **保留全部功能**：配置、多开、保存、删除、路径设置全部保留
4. **代码架构优化**：清晰分层，便于后续维护

## 四、用户操作流程

### 首次使用
```
搜索"微信多开" → 配置中心 Tab → 下载 handle → 设路径 → 切到"我的账号"Tab → 新建多开 → 扫码 → 保存
```

### 日常使用（最短路径）
```
搜索"微信多开" → 点击账号卡片 → 自动切换登录 ✅
```

### 保存新账号
```
"我的账号"Tab → 新建多开 → 扫码登录 → 点击"保存当前" → 出现在列表中
```

### 删除账号
```
右键账号卡片 → 删除 → 确认 → 从列表移除
```

### 5.2 plugin.json 变化

**旧版：6 个 feature**
```json
"features": [
  { "code": "config" },
  { "code": "wechat_start" },
  { "code": "wechat_list" },
  { "code": "wechat_delete_list" },
  { "code": "wechat_save" },
  { "code": "wechat_file_path" }
]
```

**新版：1 个 feature**
```json
"features": [
  {
    "code": "wechat_plus",
    "explain": "微信多开 Plus - 一个界面管理所有微信账号",
    "cmds": [
      "微信多开",
      "wxdk",
      "多开",
      "wechat"
    ]
  }
]
```

### 5.3 preload.js 架构变化

**旧版：6 个独立 exports**
```js
window.exports = {
  "wechat_list": { mode: "list", args: { ... } },
  "wechat_start": { mode: "none", args: { ... } },
  "wechat_save": { mode: "none", args: { ... } },
  "wechat_delete_list": { mode: "list", args: { ... } },
  "wechat_file_path": { mode: "none", args: { ... } },
  "config": { mode: "list", args: { ... } },
}
```

**新版：1 个 exports，内部路由**
```js
window.exports = {
  "wechat_plus": {
    mode: "list",
    args: {
      enter: async (action, callbackSetList) => {
        // 初始化：检测配置状态 + 加载账号列表 + 合并渲染
      },
      search: async (action, searchWord, callbackSetList) => {
        // 搜索过滤账号
      },
      select: async (action, itemData, callbackSetList) => {
        // 根据 itemData.type 路由：
        //   "account"   → 切换登录
        //   "new"       → 新建多开
        //   "save"      → 保存当前
        //   "refresh"   → 刷新状态
        //   "download"  → 下载 handle
        //   "setpath"   → 设置路径
        //   "delete"    → 删除账号
      }
    }
  }
}
```

### 5.4 index.html 变化

**旧版**：`index.html` 基本为空（仅一行 script）

**新版**：完整的双 Tab 管理界面
- Tab 1「我的账号」：快捷操作栏 + 账号卡片列表 + 底部操作按钮
- Tab 2「配置中心」：handle 状态 + 路径设置 + 使用须知
- 纯 HTML/CSS，通过 preload.js 中的 utools API 渲染数据
- 搜索框仍支持关键词过滤

### 5.5 核心层 (lib/) 保持不变

`wechatHelp.js`、`kill.js`、`file.js` 等核心逻辑不做大幅改动，仅做以下优化：
- 已修复的 bug 全部保留（路径分隔符、回调错误、进程冲突等）
- 新增 `isWeixinRunning()` 和 `killWeixinProcess()`
- 可能新增几个辅助方法供 UI 层调用

## 六、搜索关键词映射

用户可以通过以下任意关键词触发插件：

| 关键词 | 效果 |
|--------|------|
| 微信多开 | 打开主界面 |
| wxdk | 打开主界面 |
| 多开 | 打开主界面 |
| wechat | 打开主界面 |

旧版的 `wxok`、`多开确认`、`删除多开列表` 等关键词不再需要，操作全部在主界面完成。但可以考虑保留 `wxok` 作为快捷保存的触发词（可选）。

## 七、兼容性

- **平台**：仅 Windows（与现有版本一致）
- **微信版本**：仅 4.0+（与现有版本一致）
- **uTools 版本**：需要支持 `preload.js` 的版本
- **数据迁移**：`plugin_save_config` 目录结构不变，旧版保存的账号数据完全兼容
